## 1. Project Overview

## 4. Data Model

### 4.1 IndexedDB Tables (Dexie, 8 tables)

| Table          | Primary Key       | Key Indexes                                  |
|----------------|-------------------|----------------------------------------------|
| `animals`      | `animal_id`       | arete_id, nombre, tipo, sexo, estado, _sync_status |
| `health`       | `salud_id`        | animal_id, fecha, tipo_evento, _sync_status |
| `reproduction` | `reproduccion_id` | vaca_id, semental_id, fecha_monta, _sync_status |
| `observations` | `observacion_id`  | animal_id, fecha, _sync_status              |
| `sales`        | `venta_id`        | animal_id, fecha_venta, _sync_status        |
| `recorridos`   | `++_rowId` (auto) | recorrido_id, fecha, animal_id, _sync_status |
| `photos`       | `photo_id`        | animal_id, _sync_status                     |
| `users`        | `user_id`         | nombre                                       |

### 4.2 Google Sheets Worksheets (7 tabs)

| Internal Name  | Sheet Tab Name |
|----------------|----------------|
| animals        | Registro       |
| health         | Salud          |
| reproduction   | Reproduccion   |
| observations   | Observaciones  |
| sales          | Ventas         |
| recorridos     | Recorridos     |
| users          | Usuarios       |

### 4.3 Sync Metadata (all records)

Every record carries `SyncMeta`:
- `_sync_status`: `"synced"` | `"pending"` | `"conflict"` (IndexedDB only)
- `updated_at`: ISO 8601 timestamp
- `created_at`: ISO 8601 timestamp
- `created_by`: user_id string

### 4.4 Key Types (Spanish enum values)

```typescript
AnimalTipo: "Semental" | "Becerro(a)" | "Vaquilla" | "Vaca" | "Torete"
Sexo: "Macho" | "Hembra"
Temperamento: "Normal" | "Manso(a)" | "Bravo(a)"
EstadoAnimal: "Vivo(a)" | "Muerto(a)" | "Vendido(a)"
TipoEventoSalud: "Vacuna" | "Desparasitación" | "Vitamina" | "Enfermedad" | "Tratamiento" | "Revisión"
EstadoGeneral: "Fuerte" | "Flaco" | "Enfermo"
PrenezEstado: "Sí" | "No" | "Pendiente"
MotivoVenta: "Por peso (destete)" | "Por edad" | "Por productividad" | "Otro"
```

### 4.5 ID Generation

- **animal_id**: Built as `{nombre}-{arete_id}` (e.g., `Luna-A042`)
- **Other IDs**: Sequential with prefix via `generateId()`:
  `SAL-001`, `REP-001`, `OBS-001`, `VTA-001`, `REC-001`
- **User ID**: From `localStorage` key `livestock_user_id`, defaults to `USR-001`

## 5. API Endpoints

Base URL: `VITE_API_URL` env var (default `http://localhost:8000`)

| Method | Path                         | Purpose                          |
|--------|------------------------------|----------------------------------|
| GET    | `/`                          | Health check                     |
| GET    | `/health`                    | Detailed health check            |
| POST   | `/api/sync/{table_name}`     | Push pending records, get merged |
| GET    | `/api/sync/pull`             | Pull all data from Google Sheets |
| POST   | `/api/photos/upload`         | Upload single photo to Drive     |
| POST   | `/api/photos/upload/batch`   | Upload multiple photos to Drive  |
| DELETE | `/api/photos/{photo_id}`     | Delete photo from Drive          |

**Valid sync tables**: `animals`, `health`, `reproduction`, `observations`, `sales`, `recorridos`

### Sync Strategy
- **Last-write-wins** based on `updated_at` timestamp
- Client pushes `_sync_status = "pending"` records
- Server merges with Google Sheets data
- Server returns full merged dataset
- Client replaces local table with merged data, marks all `synced`

## 7. Key Patterns & Conventions

### Frontend
- **State**: No global state manager — Dexie `useLiveQuery` handles reactivity
- **Forms**: Local `useState` for form state, write to Dexie on submit
- **Offline**: All writes go to IndexedDB first with `_sync_status: "pending"`
- **Sync**: Manual via `SyncButton` calling `syncAll()` (photos first, then tables)
- **Router**: Flat route list under `RootLayout`, no nested route groups
- **Imports**: Barrel exports from `components/ui/index.ts`

### Backend
- **Services layer**: `sheets.py` (Google Sheets CRUD) + `drive.py` (photo storage)
- **Merge logic**: In `sheets.merge_records()` — reads sheet, compares timestamps, writes back
- **Error handling**: Graceful fallback when Google credentials are unavailable
- **CORS**: Allows localhost:5173/4173 + production Vercel domain

### Naming
- **Files**: PascalCase for React components, camelCase for utilities/hooks
- **Variables/Functions**: camelCase (TS), snake_case (Python)
- **Data fields**: snake_case everywhere (matches Google Sheets headers)
- **UI labels**: Spanish (e.g., "Registrar Animal", "Sincronizar")

## 11. Important Design Decisions

1. **No global state**: Dexie's `useLiveQuery` replaces Redux/Zustand —
   every component reads directly from IndexedDB reactively.
2. **animal_id format**: Built as `{nombre}-{arete_id}` (not sequential),
   so renaming an animal changes its ID.
3. **Recorridos use auto-increment**: `++_rowId` because multiple rows
   share the same `recorrido_id` (one per animal seen in a roundup).
4. **Photos stored as base64**: In IndexedDB until synced, then Drive URL
   replaces the blob. Compression: 800px max, JPEG 70%.
5. **Small dataset assumption**: <200 animals — full table sync is acceptable,
   no pagination or delta-sync needed.
6. **Google Sheets as DB**: Familiar to the ranch owner, serves as both
   database and exportable spreadsheet.
